TRAE SOLO CN

桌面端保姆级教程

------ More Than Coding，AI 原生工作台全指南 ------

独立桌面应用·三端联动·双模式

适用于开发者、产品经理、运营、数据分析师等全场景

版本：0.1.8 | 2026 年 5 月

第一章：认识 TRAE SOLO

1.1 什么是 TRAE SOLO

TRAE SOLO 是字芃跳动推出的 AI 原生工作台，是一款 独立于 TRAE IDE 运行的桌面应用程序。它不仅仅是一个编程工具，更是一个覆盖从专业开发到日常办公的全场景 AI 协作平台。

核心特点：

  • 独立架构：不依赖 TRAE IDE 运行，专为 AI Agent 设计的独立应用
  • 双模式：MTC（More Than Coding）模式 + Code 模式，适合不同背景用户
  • 三端联动：网页版、桌面版、移动版共享同一套账号体系和任务数据
  • 多元输入：支持文字、语音、附件、技能等多种输入方式
  • 实时进度：实时展示任务当前进度并自动总结输出内容
  • 深度优化：独立运行带来更强的性能和更丰富的交互拓展

1.2 三端联动：网页版、桌面版与移动版

端态 特点 适用场景
网页版 轻量便捷，无需下载安装，开箱即用 临时需求、外出办公、快速验证
桌面版 独立架构，性能全面升级，支持本地+云端任务 长期开发、复杂项目、深度工作
移动版 随时随地下发管理任务，支持语音输入 碎片化场景、跨设备任务管理

三端联动特性：

  • 移动版：跨设备的任务下发中心，适合在碎片化场景下快速下发和管理任务
  • 网页版 & 桌面版：负责任务执行、深度交互和产物验收
  • 设备离线时，系统会自动切换至云端执行任务，确保任务不中断
  • 任务状态会在三端之间无缝同步

1.3 双模式：MTC 模式 vs Code 模式

TRAE SOLO 提供两种模式，分别面向不同背景的用户：

模式 面向用户 适用场景 典型任务
MTC 模式 非开发相关用户（产品经理、数据分析师、运营等） 处理文档、数据、演示稿等 竞品调研、数据分析报告、内容创作、文件整理
Code 模式 习惯于智能体主导开发的开发工程师 编码、调试、代码库管理和 Git 工作流 前端页面开发、输出 PRD、网页小游戏、代码重构

模式选择建议

如果你是产品经理、运营、数据分析师等非开发岗位，推荐使用 MTC 模式；如果你是开发者，推荐使用 Code 模式。

1.4 与 TRAE IDE 的区别

很多人第一次听到 TRAE，会误以为 TRAE SOLO 就是 TRAE IDE 中的 SOLO 模式。其实它们是 两个独立的产品：

特性 TRAE IDE TRAE SOLO 桌面版
产品定位 AI 原生 IDE（编程工具） AI 原生工作台（全场景工具）
运行方式 集成在 IDE 中 独立桌面应用
模式 Builder/Chat/SOLO 模式 MTC/Code 双模式
目标用户 以开发者为主 开发者 + 非开发人员
三端联动 无 网页版/桌面版/移动版同步
口号 The Real AI Engineer More Than Coding

关键区别

TRAE IDE 是一个编程 IDE，而 TRAE SOLO 是一个独立的 AI 工作台。TRAE SOLO 的能力在原有 TRAE IDE SOLO 模式的基础上得到了进一步提升。

1.5 国内版 vs 国际版

特性 国内版（CN） 国际版
官网 solo.trae.cn / trae.cn solo.trae.ai / trae.ai
内置模型 豆包、DeepSeek 等国产模型 Claude、GPT 等国际模型
费用 基础功能免费 内测免费，后续可能收费
网络 无需科学上网 可能需要海外网络
登录 手机号注册（部分需邀请码） 邮箱/GitHub

推荐

国内用户直接使用国内版，免费且够用。想用 GPT 系列模型的，再考虑国际版。

第二章：安装与初始化

2.1 系统要求

平台 系统要求 备注
macOS macOS 12.0+ 支持 Apple Silicon 和 Intel 芯片
Windows Windows 10/11 目前已开放下载
Linux 暂不支持 ---

2.2 下载地址

国内版下载方式：

  1. 访问 TRAE 中国官网：https://www.trae.com.cn/
  2. 在首页找到"下载"或"SOLO 独立下载"区域
  3. 选择对应系统的安装包下载

或直接访问：

  • 国内版：https://solo.trae.cn
  • 国际版：https://solo.trae.ai
  • 下载页：https://www.trae.ai/download（找到 SOLO 下载区）

2.3 安装步骤

macOS 安装

  1. 下载 .dmg 文件（根据自己的芯片选择 Apple Silicon 或 Intel 版本）
  2. 双击打开 .dmg 文件
  3. 将 TRAE SOLO 图标拖入 Applications 文件夹
  4. 完成安装

首次打开

如果提示安全问题，请在"系统设置 > 隐私与安全性"中允许运行。

Windows 安装

  1. 下载 Windows 安装包
  2. 双击运行安装程序
  3. 按照提示完成安装（一键下一步即可）

2.4 首次启动与账号注册

  1. 打开 TRAE SOLO，点击"开始"
  2. 选择界面主题（深色/浅色）
  3. 选择"个人用户"
  4. 使用手机号注册登录（国内版直接发验证码）

邀请码

目前国内版部分时期需要邀请码才能注册。每天开通 200 个 SOLO 中国版资格，可关注官方社区获取。

2.5 界面布局介绍

TRAE SOLO 采用 三栏式布局：

区域 位置 功能
任务管理面板 左侧 管理和切换多个 AI 任务，支持多任务并行
AI 对话面板 中间 与 AI 交流的主要区域，支持文字、语音、附件输入
工具面板 右侧 包含代码编辑器、终端、浏览器预览、文档四个工具视图

这个布局与 TRAE IDE 的最大区别在于：原来你是中心，现在 AI 是中心，你在边上看它干活。

第三章：MTC 模式详解

3.1 MTC 模式定位与适用人群

MTC（More Than Coding）模式是 TRAE SOLO 的核心亮点，专为 非开发相关用户 设计，包括：

  • 产品经理
  • 数据分析师
  • 运营人员
  • 市场运营
  • 项目经理
  • 任何需要处理文档、数据、演示稿的人员

3.2 界面布局与交互

MTC 模式采用三栏式布局：

  • 左侧：任务管理面板，用来管理和切换多个 AI 任务
  • 中间：AI 对话面板，你和 MTC 交流的主要区域
  • 右侧：工具面板，包含代码编辑器、终端、浏览器预览、文档这四个工具视图

3.3 多元输入

MTC 模式支持多种输入方式：

  • 文字输入：直接在对话框中输入自然语言需求
  • 语音输入：按住说话按钮，无需打字即可下达指令
  • 附件上传：支持上传图片、PDF、Excel、Word 等多种格式文件
  • 技能调用：通过 / 命令调用已安装的 Skills

3.4 任务管理与多任务并行

MTC 模式支持 多任务并行，这是 SOLO 最被低估的功能之一：

  • 在任务执行过程中会自动新建子任务
  • 每个子任务都可以配一个独立的智能体
  • 多个任务同时跑，互不干扰
  • 你随时点左侧任务栏切换，看每个任务的进度

举例：

  • 子任务 1：在做前端页面
  • 子任务 2：在优化已有接口代码
  • 子任务 3：另一个智能体在写单元测试
  • 三件事同时推进，最后提供完整的产物汇总

3.5 实战场景

场景一：竞品调研

痛点：手动搜索、逐一体验、信息孤岛、手动整理

提示词模板：

深度分析 A 产品（Cursor）的核心功能、用户体验及商业模式，

并寻找 3-5 个市场定位相似的竞品，从以下维度进行对比分析：

- 核心功能

- 目标用户

- 盈利模式

- 近期版本迭代亮点

最终，请总结 A 产品的核心优势、潜在风险，并给出产品优化的初步建议。

输出：完整的竞品分析报告（Word 文档）

场景二：数据分析

痛点：数据分析公式不熟悉、图表制作繁琐、口径不一致

提示词模板：

请帮我分析这份数据并生成分析报告。

分析需求：

1. 按照不同的反馈类型进行归类，给到数量和占比

2. 反馈数量大于等于10个的属于高频反馈，请帮我标注出来

3. 根据用户的反馈描述情绪，按照重要性程度对用户反馈排列

可视化要求：

1. 反馈类型按照柱形图、饼图展示

2. 高频提到的需求按照词云给我展示

3. 按照表格展示后续急需解决的重点需求

输出形式：请输出一份带图表的分析报告 markdown 文档。

输出：带图表的数据分析报告

场景三：文件批量处理

痛点：效率低、准确性差、难批量化

提示词模板：

请整理"Downloads"下面的文件夹，并执行以下操作：

1. 创建分类文件夹：

- 文档（.pdf, .doc, .docx, .txt）

- 图片（.jpg, .png, .gif, .webp）

- 安装包（.dmg, .pkg, .zip）

- 其他

2. 将文件移动到对应分类文件夹

3. 在"文档"文件夹内，按年月创建子文件夹（基于文件修改日期）

4. 生成一个报告文件"整理记录.txt"，包含：

- 总文件数

- 每个分类的文件数量

- 最早和最新的文件日期

请先告诉我找到多少个文件，每个分类预计有多少，等我确认后再执行移动操作。

场景四：论文学习

痛点：检索低效、阅读门槛高、关联梳理难

提示词模板：

请帮我根据这5篇文章整理论文综述，要求易懂易理解。

[PDF01][PDF02][PDF03][PDF04][PDF05]...

输出：论文综述文档

场景五：学习备考

痛点：资料冗长、方向不清、整理耗时

提示词模板：

请根据以下资料帮我生成模拟题：

【输入】

根据 TRAE SOLO 的整体介绍页面 https://docs.trae.cn/solo/

帮助出一些题目，用于考验我自己是否了解这款产品

【出题要求】

1. 总共 50 道题：30 道单选 + 10 道多选 + 10 道简答

2. 难度分布：简单 30% / 中等 50% / 困难 20%

3. 覆盖所有内容，出题比例均匀分配

4. 每题附详细解析和对应知识点出处

【输出】

1. 模拟试卷 markdown 文档（无答案版，可打印）

2. 答案与解析（含逐题详解）

3. 知识点覆盖统计表

输出：模拟试卷、答案与解析、知识点覆盖统计表

场景六：新媒体运营

痛点：热点时效短、多平台适配繁琐、选题瓶颈

提示词模板：

请帮我完成本周的新媒体内容策划：

【账号信息】

账号定位：[官方产品宣传账号，一款面向所有人的 AI 编程产品]

目标受众：[25-35 岁互联网从业者]

运营平台：公众号、小红书、抖音、微博

【任务内容】

1. 热点调研：搜索本周与「TRAE、AI Coding、Skill、Harness」相关的热点话题 Top 5

2. 选题推荐：结合账号定位，推荐 3 个选题方向（附上你的选题理由和预期热度）

3. 对第 1 个选题，生成以下内容：

- 公众号版：标题（5 选 1） + 正文（1500-2000 字，含小标题分段）

- 小红书版：标题 + 正文（300 字以内，口语化，含 emoji 和话题标签）

- 抖音脚本版：开头 hook + 正文脚本 + 结尾引导（60 秒以内）

- 微博版：140 字以内 + 话题标签

【输出】

生成 markdown 文档，按平台分章节排列，方便逐平台复制发布。

输出：多平台内容策划文档

场景七：内容创作

请帮我写一篇关于「how to use TRAE SOLO」的教程文章，

3. 每个步骤配有截图说明（用文字描述截图位置）

4. 结尾加上 3 个实用小技巧

场景八：运营工具搭建

痛点：依赖开发团队、交付周期慢、自助能力缺失

提示词模板：

帮我开发一个黑客松倒计时器，用 HTML、CSS、Javascript 技术栈搭建，

js 交互逻辑就是显示距离结束还有多长时间。

要求：

1. 有实时倒计时

2. 需要设置倒计时时长

第四章：Code 模式详解

4.1 Code 模式定位与适用人群

Code 模式面向 习惯于智能体主导开发的开发工程师，聚焦于：

  • 编码
  • 调试
  • 代码库管理
  • Git 工作流

4.2 本地任务 vs 云端任务

特性 本地任务 云端任务
运行环境 本地计算机 云端沙箱容器
代码存储 本地文件系统 GitHub 仓库
适用场景 本地开发、调试 无需本地环境、团队协作
网络要求 无特殊要求 需要稳定网络

4.3 云端智能体（Cloud Agent）

云端智能体能让你在云端环境中执行代码分析、运行和调试等任务。其特点：

  • 统一的运行时与依赖管理
  • 稳定且隔离的远程环境
  • 避免本地环境差异带来的兼容性问题
  • 让你更专注于任务本身

4.4 GitHub 集成

TRAE SOLO 支持与 GitHub 深度集成，可以：

  • 从 GitHub 拉取项目到云端运行
  • 自动管理代码库
  • 支持 Git 工作流

4.5 实战场景

场景一：画原型图/前端页面还原

提示词模板：

请帮我参考这个前端社区页面，直接帮我还原为可交互的前端代码，

支持我直接预览。

要求：

1. 左侧类别和标签都加上灰色底色，每个名称前面都加上的表情符号

2. 点击登录，支持抖音和手机号注册登录

3. 发帖参考[图片]的样式进行设计

输出：可交互的前端页面，支持浏览器预览

场景二：输出 PRD

提示词模板：

请分析这个项目中的前端代码，为其中的"用户登录"和"用户名"两个流程，

按照以下格式生成 PRD 交互说明：

1. 前置条件：触发该流程需要满足的条件

2. 交互流程：以用户操作步骤的形式描述

3. 页面元素说明：表格形式，包含元素名称、类型、默认状态、交互规则

4. 异常流程：各种错误场景下的系统响应和提示文案

输出：PRD 交互说明文档

场景三：网页小游戏开发

对于复杂任务，建议先走 Plan 模式：

提示词模板：

我要做一个双人对战消消乐网页游戏。

请先出一份开发规划方案，不用写代码，

我要看清楚技术选型、页面结构、游戏核心逻辑，

确认之后再开始开发。

输出：开发规划方案→ 确认后执行开发 → 可运行的游戏

第五章：高级功能配置

5.1 Skills 技能系统

Skills 是一份给 AI 看的"能力说明书"。你把某个工作流程、团队规范、输出要求写进一个 Skill，以后每次触发相关任务，SOLO 会自动加载这份说明书。

Skills 的结构

skill-name/

├── SKILL.md # （必须）智能体的核心指令

├── examples/ # （可选）输入/输出示例

├── templates/ # （可选）可复用的模板

└── resources/ # （可选）参考文件、运行脚本或素材

SKILL.md 文件格式

---

name: 技能名称

description: 简要描述这个技能的功能和使用场景

---

# 技能名称

## 描述

描述这个技能的作用。

## 使用场景

描述触发这个技能的条件。

## 指令

清晰的分步说明，告诉智能体具体怎么做。

## 示例 (可选)

输入/输出示例，展示预期效果。

创建技能

方式一：用自然语言让 AI 帮你生成

帮我创建一个 Skill，用于生成标准化的竞品分析报告，

步骤是：第一步收集竞品基本信息，

第二步对比功能列表，

第三步分析用户口碑，

第四步给出差异化建议，

输出格式为 Markdown，每个部分要有二级标题。

方式二：从技能市场安装

  1. 在左侧任务栏点击"技能"
  2. 在"技能市场"页签搜索所需技能
  3. 点击技能卡片右侧的 + 按钮安装

调用技能

手动调用：在对话框中输入 / ，然后从列表中选取所需技能

自动调用：AI 会根据任务内容与技能中的"使用场景"自动判断是否调用

5.2 Rules 规则

Rules 与 Skills 的区别：

  • Rules：全量加载，一旦开启对话，所有规则都会被注入并持续占用上下文窗口
  • Skills：按需加载，仅在实际需要被调用时才加载到上下文中

适用场景：

  • 使用 Rules：需要始终生效的全局规范（如"所有注释用中文"）
  • 使用 Skills：特定场景的工作流（如"生成竞品分析报告"）

5.3 内置工作流：Spec & Plan 模式

对于复杂任务，建议先走 Plan 模式：

  1. 在开始任务前，明确告诉 SOLO：先不要写代码，先输出方案
  2. SOLO 会生成：需求拆解、任务顺序、技术选型、风险点
  3. 你在方案上确认、修改、拒绝某些步骤
  4. 确认后，SOLO 才开始执行

经验值

对于超过 10 个文件改动的任务，先走 Plan 模式，最终节省的时间远超过规划本身花掉的时间。

5.4 沙箱安全机制

沙箱为智能体生成的命令提供受限的执行环境，确保智能体在安全隔离的环境中运行命令。

三种命令运行模式：

  • 沙箱运行（默认）：命令在安全沙箱中自动运行并判定风险
  • 手动运行：所有命令需手动确认
  • 自动运行：所有命令自动执行

安全提示

删除文件或代码等危险操作，AI 会在执行前寻求用户批准。建议新手使用默认的沙箱运行模式。

5.5 上下文压缩

用 SOLO 跑复杂任务，对话会越来越长。上下文太长，模型容易失焦。

解决方案：

  • 在对话面板顶部有一个上下文使用量的显示条
  • 当上下文接近上限时，点击压缩按钮
  • TRAE 会自动把历史对话精炼成摘要，保留关键信息，释放上下文空间

什么时候该压缩：

  • 跑了很长时间的任务，超过 20 轮对话
  • 发现 SOLO 的回答开始重复或偏离方向
  • 要开始一个新的子阶段任务前

第七章：常见问题与资源

7.1 常见问题 FAQ

Q1：TRAE SOLO 和 TRAE IDE 有什么区别？

TRAE SOLO 是一个独立的 AI 工作台应用，不依赖 TRAE IDE 运行。TRAE IDE 是一个编程 IDE，主要面向开发者；而 TRAE SOLO 是一个全场景的 AI 工作台，面向开发者和非开发人员。

Q2：如何选择 MTC 模式还是 Code 模式？

如果你是产品经理、运营、数据分析师等非开发岗位，推荐使用 MTC 模式；如果你是开发者，推荐使用 Code 模式。

Q3：国内版需要邀请码吗？

目前国内版部分时期需要邀请码。每天开通 200 个 SOLO 中国版资格，可关注官方社区获取。

Q4：如何切换模式？

点击界面左上角的模式切换按钮，在 Code 和 MTC 之间切换。

Q5：三端数据如何同步？

网页版、桌面版、移动版共享同一套账号体系和任务数据，任务状态会在三端之间无缝同步。
